## Enforcement Implementation Notes

### Resource Counting Queries Needed

For each numeric limit, you need a query to count current usage:

```typescript
// Example: Count contacts for an organization
export const countContacts = query({
  args: { organizationId: v.id("organizations") },
  handler: async (ctx, args) => {
    const contacts = await ctx.db
      .query("objects")
      .withIndex("by_organization_and_type", (q) =>
        q.eq("organizationId", args.organizationId).eq("type", "crm_contact")
      )
      .collect();
    return contacts.length;
  },
});
```

### Enforcement Check Pattern

Before any create operation, check limits:

```typescript
// Example: Before creating a contact
export const createContact = mutation({
  args: { /* ... */ },
  handler: async (ctx, args) => {
    const license = await getLicense(ctx, args.organizationId);
    const currentCount = await countContacts(ctx, args.organizationId);
    
    if (currentCount >= license.crm.maxContacts) {
      throw new Error("Contact limit reached. Upgrade to add more contacts.");
    }
    
    // Proceed with creation...
  },
});
```

// Example: Check if feature is enabled
